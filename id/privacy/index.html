Kebijakan Privasi

Cardio Analytics berkomitmen melindungi data kesehatan Anda dengan privasi berdasarkan desain

Privasi Berdasarkan Desain

Cardio Analytics dibangun dengan privasi sebagai prioritas utama. Data kesehatan Anda sangat sensitif, dan kami percaya Anda harus memiliki kontrol penuh atas data tersebut.

Prinsip Inti Kami

  • Pemrosesan lokal - Semua data disimpan dan diproses di perangkat Anda
  • Tidak ada transmisi cloud - Data kesehatan Anda tidak pernah meninggalkan iPhone Anda
  • Tidak ada pelacakan - Tidak ada SDK analitik, tidak ada pelacak iklan
  • Tidak diperlukan akun - Tidak ada login, tidak ada kata sandi, tidak ada pendaftaran server
  • Kode sumber terbuka - Implementasi privasi kami dapat diaudit oleh siapa saja

Penyimpanan Data

Semua data kesehatan Anda disimpan secara lokal di iPhone Anda menggunakan kerangka kerja Apple HealthKit dan penyimpanan perangkat iOS standar:

Di Mana Data Anda Disimpan

  • Apple HealthKit - Metrik kardiovaskular (detak jantung, tekanan darah, HRV, SpO₂, ECG, dll.) disimpan di database HealthKit terenkripsi Apple
  • Penyimpanan lokal aplikasi - Entri obat, log gejala, dan preferensi aplikasi disimpan di sandbox aplikasi Cardio Analytics
  • iCloud (opsional) - Jika Anda mengaktifkan Cadangan iCloud untuk aplikasi ini, pengaturan aplikasi (BUKAN data HealthKit) dapat dicadangkan ke iCloud

Enkripsi & Keamanan

  • Data HealthKit dilindungi oleh enkripsi perangkat Apple standar
  • Data hanya dapat diakses saat perangkat Anda tidak terkunci (Perlindungan Data Lengkap)
  • Cadangan dienkripsi jika Anda menggunakan cadangan terenkripsi iTunes/Finder atau iCloud

Data yang Kami Kumpulkan

Cardio Analytics HANYA mengakses data yang Anda otorisasi secara eksplisit melalui prompt izin HealthKit:

Data Kesehatan (dengan Izin Anda)

  • Metrik kardiovaskular - Detak jantung, tekanan darah, HRV, SpO₂, ECG, notifikasi ritme tidak teratur
  • Metrik kebugaran - VO₂ Max, kecepatan berjalan, asimetri berjalan, kecepatan naik tangga
  • Metrik tubuh - Berat badan, BMI
  • Obat - Rejimen obat (jika diotorisasi)

Data yang Anda Masukkan

  • Informasi obat - Nama obat, dosis, waktu (disimpan secara lokal)
  • Gejala - Entri gejala dengan cap waktu dan tingkat keparahan (disimpan secara lokal)
  • Tujuan perawatan - Target metrik yang Anda tetapkan (disimpan secara lokal)
  • Ambang peringatan kustom - Rentang normal yang dipersonalisasi (disimpan secara lokal)

Data yang TIDAK Kami Kumpulkan

  • ❌ Informasi identitas pribadi (nama, email, nomor telepon)
  • ❌ Data lokasi
  • ❌ Daftar kontak
  • ❌ Foto atau kamera
  • ❌ Aktivitas penelusuran atau riwayat pencarian
  • ❌ Pengenal iklan atau perangkat

Berbagi Data

Cardio Analytics TIDAK PERNAH membagikan data kesehatan Anda dengan pihak ketiga. Anda memiliki kontrol penuh:

Tidak Ada Berbagi Otomatis

  • Tidak ada data kesehatan yang dikirim ke server kami
  • Tidak ada data yang dijual kepada broker data atau pengiklan
  • Tidak ada berbagi dengan penyedia layanan pihak ketiga
  • Tidak ada penelitian atau agregasi data

Berbagi Atas Pilihan Anda

Anda dapat berbagi data kesehatan HANYA ketika Anda secara eksplisit memilih untuk melakukannya:

  • Ekspor PDF/CSV - Buat laporan dan bagikan secara manual dengan dokter Anda via email atau AirDrop
  • Bagikan Lembar iOS - Gunakan dialog berbagi sistem iOS standar untuk mengirim laporan
  • Kontrol penuh - Anda memutuskan kapan, kepada siapa, dan data apa yang dibagikan

Izin HealthKit

Cardio Analytics meminta izin terperinci untuk setiap jenis data HealthKit yang diaksesnya:

Kontrol Terperinci

  • Anda dapat memberikan atau menolak akses ke setiap jenis data individual (detak jantung, tekanan darah, dll.)
  • Anda dapat mencabut izin kapan saja melalui Pengaturan iOS → Privasi → Kesehatan
  • Aplikasi akan terus berfungsi dengan subset data yang Anda pilih untuk dibagikan

Izin Baca vs Tulis

  • Izin baca - Memungkinkan Cardio Analytics untuk melihat data yang ada di HealthKit
  • Izin tulis - Memungkinkan Cardio Analytics untuk menulis data baru ke HealthKit (misalnya, entri tekanan darah manual, obat)
  • Anda dapat memberikan izin baca tanpa izin tulis, atau sebaliknya

Analitik & Diagnostik

Cardio Analytics TIDAK menggunakan analitik atau pelacak pihak ketiga. Namun, informasi crash diagnostik dasar dapat dikumpulkan melalui sistem standar Apple:

Laporan Crash Apple

  • Jika Anda mengaktifkan "Bagikan Analitik iPhone & Apple Watch" di Pengaturan iOS, laporan crash aplikasi DAPAT dibagikan dengan Apple dan pengembang aplikasi
  • Laporan crash ini TIDAK mengandung data kesehatan—hanya informasi teknis tentang crash aplikasi
  • Anda dapat menonaktifkan ini di Pengaturan → Privasi → Analitik & Peningkatan

Tidak Ada Pelacakan Perilaku

  • Kami TIDAK melacak bagaimana Anda menggunakan aplikasi
  • Kami TIDAK mengumpulkan metrik "keterlibatan" atau "retensi"
  • Kami TIDAK menggunakan SDK analitik (Firebase, Mixpanel, dll.)

Kepatuhan Hukum

HIPAA & Regulasi Kesehatan

Meskipun Cardio Analytics adalah aplikasi konsumen (bukan sistem catatan kesehatan elektronik), kami mengikuti praktik terbaik yang selaras dengan prinsip HIPAA:

  • Penyimpanan data lokal (tidak ada transmisi server)
  • Enkripsi perangkat
  • Tidak ada berbagi tanpa persetujuan eksplisit
  • Akses terkontrol (izin HealthKit terperinci)

GDPR & Privasi Data

Untuk pengguna di Uni Eropa:

  • Anda memiliki hak akses, portabilitas, dan penghapusan data Anda
  • Karena semua data disimpan secara lokal di perangkat Anda, ANDA mengontrol sepenuhnya
  • Menghapus aplikasi menghapus semua data aplikasi (data HealthKit tetap ada kecuali dihapus melalui Apple Health)

Privasi Anak-Anak

Cardio Analytics tidak ditujukan untuk anak-anak di bawah usia 13 tahun. Kami tidak secara sengaja mengumpulkan informasi pribadi dari anak-anak. Jika Anda adalah orang tua dan percaya anak Anda telah memberikan informasi pribadi kepada kami, silakan hubungi kami.

Perubahan pada Kebijakan Ini

Kami dapat memperbarui Kebijakan Privasi ini dari waktu ke waktu. Kami akan memberi tahu Anda tentang perubahan apa pun dengan memposting kebijakan baru di halaman ini dan memperbarui tanggal "terakhir diperbarui" di bagian atas.

Anda disarankan untuk meninjau Kebijakan Privasi ini secara berkala untuk setiap perubahan. Perubahan pada Kebijakan Privasi ini efektif ketika diposting di halaman ini.

Hubungi Kami

Jika Anda memiliki pertanyaan tentang Kebijakan Privasi ini atau praktik privasi kami, silakan hubungi kami:

  • Email: info@onmedic.com
  • Formulir dukungan: Halaman Dukungan

✅ Tidak diperlukan akun
